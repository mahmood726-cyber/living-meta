# Limitations and Known Issues

## Scope Limitations

### Data Source Constraints
- **Registry-Only**: This tool exclusively uses ClinicalTrials.gov registry data. It cannot access:
  - Full-text publications
  - Individual participant data (IPD)
  - Regulatory submissions
  - Conference abstracts or gray literature

### Outcome Data Limitations
- **Aggregate Data Only**: All analyses use aggregate summary statistics from registry postings
- **Limited Timepoints**: Registry results may not include all clinically relevant timepoints
- **Selective Reporting**: Registered outcomes may differ from those ultimately reported
- **Missing Primary Outcomes**: Some trials post results without primary outcome data

### Effect Size Constraints
- **Binary Outcomes**: Limited to events and denominators (no rate data)
- **Continuous Outcomes**: Requires mean, SD, and N (no median/IQR handling)
- **Survival Outcomes**: HR extraction from registry is inconsistent; many trials don't post HRs

## Methodological Limitations

### Risk of Bias Assessment
- **ROB 2.0 Implementation**: Domain 3 (missing outcome data) cannot be fully assessed from registry data alone
- **Blinding Information**: Registry often lacks detailed blinding protocols
- **Protocol Deviations**: Not systematically reported in registries

### Heterogeneity Estimation
- **REML Fallback**: WASM REML estimator falls back to DerSimonian-Laird when matrix operations fail
- **Small Sample Bias**: HKSJ adjustment may be insufficient with k < 5 studies
- **Prediction Intervals**: Require k >= 3 for df calculation; uninformative with high heterogeneity

### Publication Bias Tests
- **Power Limitations**: Egger/Peters/Harbord tests require k >= 10 for adequate power
- **Non-Publication ≠ Publication Bias**: Registry non-publication is distinct from publication bias in traditional sense

### Trial Sequential Analysis
- **Information Fraction**: RIS calculations assume homogeneous effect across studies
- **Boundary Approximations**: O'Brien-Fleming spending function uses simplified Lan-DeMets approximation
- **Futility Boundaries**: Only calculated when information fraction > 50%

## Technical Limitations

### Browser Compatibility
- **IndexedDB Dependency**: Requires browser with IndexedDB support
- **Worker Support**: Analysis functions require Web Worker support
- **Storage Limits**: Large projects may hit browser storage quotas

### Numerical Precision
- **JavaScript Floating Point**: All JS calculations subject to IEEE 754 precision limits
- **Distribution Functions**: t, chi-square, gamma functions use approximations
- **Extreme Values**: Very small p-values (< 1e-10) may lose precision

## Evidence Integrity Module (EIM) Caveats

### Non-Publication Risk
- **Time-Based Only**: Uses completion date + 24 months threshold
- **False Positives**: Some trials may have legitimate delays (regulatory review, publication process)
- **Registry Updates**: Trials may post results after initial EIM assessment

### Outcome Switching Detection
- **Name Matching**: Uses fuzzy matching which may miss semantic equivalence
- **Protocol Amendments**: Cannot distinguish pre-specified amendments from post-hoc changes

### Coverage Assessment
- **Registry Completeness**: Assumes ClinicalTrials.gov is complete for intervention type
- **Results Posting Rate**: May not reflect actual publication rate in journals

## Validation Status

### Validated Components (vs metafor in R)
- Fixed effects pooled estimate (tolerance: 1e-6)
- DerSimonian-Laird random effects (tolerance: 1e-4)
- HKSJ-adjusted confidence intervals (tolerance: 1e-4)
- Prediction intervals (tolerance: 1e-4)
- I² and Q statistics (tolerance: 1e-6)
- Egger regression intercept (tolerance: 1e-4)
- E-values (tolerance: 1e-4)

### Partially Validated
- REML estimation (falls back to DL)
- Peters and Harbord tests (simplified implementations)
- TSA boundaries (approximation-based)

### Not Yet Validated
- Network meta-analysis (placeholder implementation)
- SUCRA rankings
- Inconsistency measures

## Reporting Requirements

All reports generated by this tool should include:

1. **Coverage Warning**: Percentage of included trials with posted results
2. **EIM Summary**: Non-publication rate, missing participant-years
3. **TSA Status**: Whether required information size is reached
4. **Validation Statement**: Reference to validation against R metafor
